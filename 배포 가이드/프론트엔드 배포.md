# 프론트엔드 배포 가이드

## 사전 조건

- Node.js 18 이상
- AWS CLI 설정
- S3 버킷 `english-agent-hub-front` 생성
- CloudFront 신규 배포 권장
- 운영 도메인 `english.dxline-tallent.com` 연결 권장

## 환경변수

EnglishAgentHub 프론트는 `NEXT_PUBLIC_API_URL`을 사용한다.

```bash
export NEXT_PUBLIC_API_URL=https://english.dxline-tallent.com
```

기존 BeautyBook의 `NEXT_PUBLIC_API_BASE_URL`과 이름이 다르므로 혼동하지 않는다.

관련 코드:

```text
english-agent-hub-front/src/shared/api/axios.ts
english-agent-hub-front/src/entities/agent/api/agentChatApi.ts
```

## 로컬 빌드

```bash
cd /Users/terecal/english-agent-hub-container/english-agent-hub-front

export NEXT_PUBLIC_API_URL=https://english.dxline-tallent.com
npm install
npm run build

ls -la out/
```

`next.config.ts`는 production에서 static export를 사용한다.

## S3 업로드

권장: BeautyBook과 다른 S3 버킷 사용. EC2와 달리 S3 버킷 추가 자체에는 서버 고정비가 붙지 않는다.

```bash
aws s3 mb s3://english-agent-hub-front --region ap-northeast-2
```

정적 파일 업로드:

```bash
aws s3 sync out/ s3://english-agent-hub-front \
  --delete \
  --region ap-northeast-2
```

## CloudFront 무효화

CloudFront 배포 생성 후 배포 ID를 기록한다.

```bash
aws cloudfront create-invalidation \
  --distribution-id YOUR_ENGLISH_AGENT_HUB_DISTRIBUTION_ID \
  --paths "/*"
```

## 같은 S3 버킷을 꼭 써야 할 때

비용 때문에 같은 S3를 쓸 필요는 거의 없다. 그래도 운영 정책상 같은 버킷을 써야 하면 아래 조건이 필요하다.

- `s3://beauty-book-hair-front/english-agent-hub/` prefix에 업로드
- `next.config.ts`의 `basePath`, `assetPrefix`를 `/english-agent-hub`로 설정
- CloudFront Function에서 `/english-agent-hub/*` static export rewrite 추가
- `aws s3 sync` 시 BeautyBook 파일을 지우지 않도록 prefix만 대상으로 sync

예:

```bash
aws s3 sync out/ s3://beauty-book-hair-front/english-agent-hub \
  --delete \
  --region ap-northeast-2
```

이 방식은 라우팅과 캐시 정책이 복잡해지므로 신규 버킷이 더 안전하다. 핵심 비용 절감은 백엔드 EC2 공유로 달성한다.
