## SSH 접속

기존 BeautyBook PEM 키를 재사용할 수 있다. 키 파일은 이 프로젝트에 복사해도 되지만 Git에 올리지 않는다.

```bash
chmod 400 "배포 가이드/hibot-d-server-key.pem"
ssh -i "배포 가이드/hibot-d-server-key.pem" ubuntu@13.209.195.64
```

## 주의사항

- 기존 `dxline-tallent.com/api/*`는 BeautyBook `4101`로 가는 경로다.
- EnglishAgentHub를 같은 도메인의 `/api/*`에 붙이면 BeautyBook API와 충돌한다.
- EnglishAgentHub는 `english.dxline-tallent.com/api/* -> localhost:4301`처럼 도메인을 분리한다.
- 비용 절감 대상은 EC2다. S3 버킷은 별도로 만들어도 저장량/요청량 기준 과금이라 이 프로젝트 규모에서는 운영 분리 이점이 더 크다.
- EC2 메모리가 2GB라서 Java 서비스 2개 + Postgres 2개 운영 시 swap 설정과 로그 관리가 필요하다.
